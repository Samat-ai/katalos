PIXEL ASSET RECIPE — NO CANVAS, NO IMAGES
Sprites = one div + box-shadow list from an ASCII map, 3–4px pixel unit; idle animation = two stacked frame divs with opposite steps(1) opacity keyframes (rare 90/10 split for blinks, 50/50 for tail swish). Scene loops: twinkling window stars + drifting clouds (stepped translateX), bouncing DVD screensaver on the idle CRT — all frozen under prefers-reduced-motion. Object recipe (matches the refs): ink outline → body fill → top-left highlight → bottom-right shade → 2–3 detail pixels (page edges, label stripe, LED). Remote covers stay photographic inside a 2–3px ink frame — the frame does the pixelating, don’t filter the art. image-rendering:pixelated only for thumbnails ≤40px.
CONTENT EDGE RULES
Long titles: 2-line clamp in drawers, 1-line ellipsis in rows, full title in the themed PixelTooltip · book/tape hover = pull-out (translateY down), never a lift · missing cover → initials block in entry color · missing rating → “no rating yet”, never empty stars · empty zones → dashed footprint + one line · zone overflow → newest 12 + “+N more” chip · every catalog failure path lands on manual entry.
